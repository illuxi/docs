## Attribution

Enrollment Access can be assigned to either a user or a group. When it is assigned to a group, any user with [membership](https://illuxi.mintlify.app/concepts/user_grp)  in that group will receive the access. Once a user takes action on the group Enrollment Access, a separate Enrollment Access is created and assigned to that user.

## Access status

An access can have different statuses throughout its lifecycle.

| Status | Description |
| --- | --- |
| Registration required | The user has access to the course but must register<br /> before starting. |
| Registration with manager approval | The user must register, and the registration must be<br />approved by their manager before access is granted. |
| Registration with admin approval | The user must register, and the registration must be<br />approved  by an administrator before access is granted. |
| Awaiting approval | The user has registered and is awaiting an approval <br />decision. |
| Registration rejected | The user’s registration request was reviewed and<br /> denied. |
| Registered | The user is registered and can now access or <br />begin the course. |
| No participation | The user had access to the course but did not start<br /> or participate. |
| Refunded | The user purchased the course but was <br />refunded and no longer has active access. |
